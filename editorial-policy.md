## 1. Общая стратегия контента

### Принцип недельных тем
**Правило:** Одна неделя = одна тема

**Пример структуры недели:**
- **Понедельник:** Теория
- **Вторник:** Задача с собеседования (привязана к теории)
- **Среда-четверг:** Частая ошибка / дополнительная теория / практика
- **Пятница:** Резюме / практическое применение
- **Выходные:** Опционально (легкий контент / отдых)

**Примеры тем:**
- Сортировки (неделя 15-21 июня)
- Хеш-мапы
- Two Pointers
- Префиксные вычисления

**Логика:** Последовательное изучение одной темы помогает аудитории глубже понять материал и сразу применить на практике.

## 2. Типы постов и их структура

### 2.1. Теория (понедельник/начало недели)

**Цель:** Дать фундаментальное понимание темы

**Структура:**
1. **Хук** (жирный заголовок + эмодзи)
   - Пример: `<strong>Что нужно знать про сортировки для успешного прохождения собесов?</strong>`

2. **Маркер сложности** (если нужно)
   - `🐥 Материал подойдет для новичков`
   - `📊 Материал среднего уровня`

3. **Основной контент** с нумерацией через эмодзи
   - `1️⃣ Первый пункт`
   - `2️⃣ Второй пункт`
   - `3️⃣ Третий пункт`

4. **Разделитель**
   - `——` (длинное тире)

5. **Анонс следующего поста**
   - Пример: "Завтра рассмотрим задачу из Озона..."

6. **CTA (Call to Action)**
   - Пример: "Ставь 👍, если было полезно"
   - Варианты реакций с пояснениями

7. **Хештеги**
   - Формат: `#теория #[тема_недели]`

**Особенности:**
- Объяснение "почему", а не только "как"
- Личный опыт и инсайты
- Ссылки на внешние ресурсы (при необходимости)

### 2.2. Задача с собеседования (вторник, привязана к теории)

**Формат:** 2 поста (условие + разбор через 3 часа)

#### Пост 1: Условие задачи (утро, 07:00-12:00)

**Структура:**
1. **Хук с эмодзи и компанией**
   - `🔥 Свежая задача из [Компания]`
   - `❗️ Очень свежая задача из [Компания]`, [месяц год]

2. **Сложность**
   - `**Сложность:** Easy / Medium / Hard`

3. **Условие задачи**
   - Четко, с примерами
   - Требования (временная/пространственная сложность)

4. **Примеры**
   - Минимум 2 примера
   - Формат: `- Вход: ... / - Выход: ...`
   - Объяснение (при необходимости)

5. **Подсказка в спойлере**
   - `**Подсказка:** <spoiler>текст подсказки</spoiler>`

6. **Ссылка на LeetCode** (если есть)
   - `Проверить решение можно [тут](ссылка)`
   - Если нет: "На LeetCode этой задачи нет (что характерно для [Компания])"

7. **Время разбора**
   - "Разбор через 3 часа!"

8. **CTA**
   - "Попробуйте решить сами - это отличная практика"
   - "Кто справится, ставьте 👨‍💻"

9. **Хештеги**
   - `#[компания_строчными] #задача`

#### Пост 2: Разбор (через 3 часа)

**Структура:**
1. **Заголовок**
   - `<strong>Разбор задачи из [Компания]</strong> из прошлого поста`

2. **Паттерн**
   - `**Паттерн:** [название паттерна]`

3. **Ключевая идея**
   - `**Ключевая идея:** [краткое объяснение]`

4. **Алгоритм** (пошагово с эмодзи)
   - `1️⃣ Шаг первый`
   - `2️⃣ Шаг второй`
   - `3️⃣ Шаг третий`

5. **Код на Python**
   - С комментариями
   - В блоке `<pre>...</pre>`

6. **Пошаговый пример** (если нужно)
   - Визуализация работы алгоритма на конкретных данных

7. **Сложность**
   - `**Сложность:** O(...) времени, O(...) памяти`

8. **Дополнительный контекст**
   - Статистика: "По моей статистике вероятность встретить..."
   - Где еще встречается задача

9. **Разделитель** (если есть дополнительный блок)
   - `———` или `***`

10. **CTA**
    - Связь с курсом/занятиями
    - Призыв к реакции

11. **Хештеги**
    - `#[компания_строчными] #разбор #[паттерн]`

## 3. Правила оформления

### 3.1. Структурирование через эмодзи

**Для нумерации:**
- `1️⃣ 2️⃣ 3️⃣ 4️⃣ 5️⃣ 6️⃣ 7️⃣ 8️⃣ 9️⃣`

**Для акцентов:**
- `🔥` — горячая/свежая новость, задача
- `❗️` — очень важно, свежая задача
- `❌` — ошибка
- `✅` — правильно, успех
- `💡` — идея, инсайт
- `📊` — статистика, данные
- `🎯` — цель, фокус
- `🐥` — материал для новичков
- `🌱` — анонс, что-то новое
- `❤️` — призыв к поддержке
- `✏️` — упражнение
- `👨‍💻` — кодинг, решение задачи
- `👀` — "хочу больше", смотрите
- `👍` — лайк, полезно

**Медали для топов:**
- `🥇 🥈 🥉`

### 3.2. Выделение текста

**Жирный:**
- Заголовки секций: `<strong>Заголовок</strong>`
- Важные термины при первом упоминании
- Выводы и ключевые моменты

**Курсив:**
- Примечания: `<em>Это контент может быть сложен для новичков</em>`
- Мета-комментарии

**Код:**
- Инлайн: `<code>переменная</code>`
- Блоки: `<pre>код</pre>`

**Спойлеры:**
- `<spoiler>скрытый текст</spoiler>`

### 3.3. Разделители

**Между секциями:**
- `——` (длинное тире) для легкого разделения
- `———` (3 длинных тире) для более сильного разделения
- `***` (альтернатива)

### 3.4. Ссылки

**Формат:**
- `<a href="URL">текст ссылки</a>`
- Для LeetCode: всегда полный URL
- Для внутренних постов: используй короткую ссылку `https://t.me/korovkina_algo/[номер]`

### 3.5. Списки

**Маркированные:**
- `•` (буллеты) для простых списков
- `-` (дефисы) для перечислений

**Нумерованные:**
- `1. 2. 3.` для последовательных шагов без эмодзи
- `1️⃣ 2️⃣ 3️⃣` для более визуальных списков

## 4. Система хештегов

### 4.1. Обязательные хештеги

**По типу контента:**
- `#теория` — теоретические посты
- `#задача` — условие задачи
- `#разбор` — разбор задачи
- `#ошибка` или `#ошибки` — частые ошибки
- `#материалы` — подборки материалов
- `#инструменты` — инструменты для подготовки
- `#статистика` — посты со статистикой

**По компаниям (строчными):**
- `#яндекс`
- `#озон`
- `#авито`
- `#вк` (не ВКонтакте)
- `#wildberries`
- `#тбанк`
- `#сбербанк`

**По темам/паттернам:**
- `#хешмапы` (одним словом)
- `#префиксы`
- `#сортировки`
- `#twopointers`
- `#бинарныйпоиск`
- `#стек`
- `#очередь`
- `#деревья`
- `#графы`
- `#рекурсия`
- `#строки`
- `#массивы`

**Общие:**
- `#алгоритмы` — общие посты
- `#собеседования` — про процесс найма
- `#подготовка` — советы по подготовке
- `#карьера` — карьерные вопросы
- `#сложность` — про Big O

### 4.2. Правила использования хештегов

1. **Минимум 2, максимум 4 хештега** на пост
2. **Обязательно:** тип контента + тема/компания
3. **Опционально:** дополнительный контекст
4. **Всегда строчными** буквами
5. **Без пробелов** внутри тега
6. **В конце поста** после CTA

**Примеры комбинаций:**
- Условие задачи: `#[компания] #задача` (всегда 2 тега)
- Разбор задачи: `#[компания] #разбор #[тема]` (2-3 тега, добавляем тему недели)
- Теория: `#теория #[тема]`
- Ошибка: `#ошибка #[тема]` или `#ошибки #теория`
- Статистика: `#статистика #собеседования #подготовка #карьера`

**Специфика для задач:**
- **Условие:** Только компания + "задача" (например: `#озон #задача`)
- **Разбор:** Компания + "разбор" + тема недели (например: `#озон #разбор #сортировки`)

## 5. Tone of Voice (стиль общения)

### 5.1. Характеристики

**Дружелюбный, но экспертный:**
- Говорю на равных, но показываю глубокие знания
- Делюсь личным опытом (Яндекс, курс, студенты)

**Неформальный:**
- "Че, пацаны и дамы, хеш-мапы?"
- "м?" в конце вопроса
- Использую "вы" и "ты" в зависимости от контекста

**Поддерживающий:**
- "Приходите ко мне, мне кайф с вами работать"
- Упоминание успехов студентов
- Позитивная мотивация

**Местами строгий:**
- "Ну ясно, практику все пропустили. А собесы как проходить без практики планируете, м? Слежу за вами."
- Подталкивание к действию

**Образовательный:**
- Объясняю сложное простым языком
- Развенчиваю мифы
- Даю глубокие инсайты, а не поверхностные факты

### 5.2. Что делаю

✅ Объясняю "почему", а не только "как"
✅ Привожу примеры из реальной практики
✅ Делюсь статистикой и данными
✅ Связываю теорию с практикой (задачи с собесов)
✅ Признаю, когда ошибалась (пост про строки)
✅ Прошу обратную связь ("пишите в комментах")
✅ Даю упражнения и подсказки

### 5.3. Что НЕ делаю

❌ Не использую сложные термины без объяснения
❌ Не даю готовые ответы без понимания
❌ Не обещаю "легких путей" к успеху
❌ Не критикую других экспертов/курсы
❌ Не продаю агрессивно (мягкие CTA)

## 8. Вовлечение аудитории

### 8.1. Призывы к действию (CTA)

**Для задач:**
- "Попробуйте решить сами"
- "Кто справится, ставьте [эмодзи]"
- "Делитесь решениями в комментариях"

**Для теории:**
- "Дайте ❤️, если такой контент интересен"
- "Ставь 👍, если было полезно"
- "Пишите, если хотите продолжение"

**Для статистики:**
- "Перешли друзьям, которые ищут работу"
- "Поддержите меня лайком, чтобы я знала, что это важно"

**Для ошибок:**
- "Напишите, если встречали эту ошибку"
- "Дайте лайк, если пост понравился"

**Когда использовать:**
- Выбор следующей компании для разбора

- Определение уровня аудитории
- Вовлечение в середине недели

**Формат:**
- Анонимные опросы
- 3-6 вариантов ответа
- Можно выбрать несколько (если уместно)
- Использую эмодзи для вариантов (когда нужно быстрое голосование)

### 8.3. Упражнения

**Типы:**
- Вопросы на понимание
- Задачи для самостоятельного решения
- "Какая сложность у этого кода?"

**Формат:**
- Четкая формулировка
- Ожидание ответов в комментариях
- Иногда разбор в следующем посте (если есть отклик)

**Назначение:**
- Сбор задач от аудитории
- Анонимный фидбек
- Реальный опыт собеседований

**Упоминание:**
- В конце разборов
- "Поделитесь своим опытом собеседования в [этой форме](ссылка)"
- "Задачи из формы я разбираю вне очереди"

---

## 9. Кросс-ссылки и навигация

### 9.1. Ссылки на прошлые посты

**Когда использовать:**
- При продолжении темы
- При упоминании ранее разобранной концепции
- Для углубления в тему

**Формат:**
- "мы говорили о [тема] [тут](ссылка)"
- "об этом уже [говорили](ссылка)"
- "помните пост про [тема]?"

**Примеры:**
- "о кваратичной сложности говорили тут: https://t.me/korovkina_algo/23"
- "глянь вчерашний пост" (без ссылки, если был вчера)

### 9.2. Анонсы следующих постов

**В конце поста:**
- "Завтра разберу свеженькую задачу из Яндекса"
- "В следующем посте расскажу про [тема]"

**Создание интриги:**
- "Задача не простая, но в посте есть подсказка 😉"
- "И это как раз тот случай, когда..."

---

## 10. Особые форматы

### 10.1. Серийные посты

**Продолжение объяснения:**
- "Еще немного о [тема]"
- "Продолжаем про [тема]"
- Нумерация не обязательна

**Многочастевые разборы:**
- Часть 1: Основы
- Часть 2: Продвинутые концепции
- Обязательна ссылка на предыдущую часть

### 10.2. Легкий контент

**Мемы:**
- С подписью, связанной с алгоритмами
- Неформальный тон
- Короткий текст

**Мотивация:**
- Короткие посты о практике
- Личные истории
- Поддержка аудитории

## 11. Качество и проверка

### 11.1. Чек-лист перед публикацией

**Обязательно проверить:**
- [ ] Хук привлекает внимание
- [ ] Структура логична (введение → тело → заключение)
- [ ] Код протестирован (если есть)
- [ ] Примеры корректны
- [ ] Сложности указаны правильно
- [ ] Нет опечаток и ошибок
- [ ] Эмодзи используются единообразно
- [ ] Хештеги соответствуют правилам
- [ ] CTA присутствует
- [ ] Форматирование корректное (жирный, код, спойлеры)

### 11.2. После публикации

**Отслеживать:**
- Реакции (❤️, 🔥, 👍, 👀)
- Комментарии
- Участие в опросах
- Переходы по ссылкам (если возможно)

**Реагировать:**
- Отвечать на вопросы в комментариях
- Исправлять ошибки, если находят
- Благодарить за конструктивную критику
- Учитывать фидбек в следующих постах

## 12. Метрики успеха

### 12.1. Количественные

**Хороший пост:**
- 5+ реакций
- 1-2 комментария
- Участие в опросе (если есть)

**Отличный пост:**
- 10+ реакций
- 3+ комментариев
- Сохранения (если видно)

### 12.2. Качественные

**Признаки успеха:**
- Вопросы в комментариях (показывает интерес)
- Благодарности
- Обсуждение темы
- Просьбы разобрать похожие задачи
- Упоминание, что задача попалась на собесе

---

## 13. Чего избегать

❌ **Слишком длинные посты** без структуры
❌ **Только код** без объяснения
❌ **Абстрактная теория** без примеров
❌ **Агрессивные продажи** курса/консультаций
❌ **Копирование** задач без добавления ценности
❌ **Негатив** про компании или других экспертов
❌ **Обещания** типа "выучите за неделю"
❌ **Пропуск подсказок** в задачах
❌ **Игнорирование фидбека** аудитории
❌ **Нерегулярность** публикаций (в активный период)

---

## 14. Эволюция политики

**Как обновлять:**
- Отслеживать, что работает лучше
- A/B тестировать форматы
- Учитывать запросы аудитории
- Смотреть на метрики
- Добавлять новые правила по мере роста

**Текущие эксперименты:**
- Добавление большего визуала (схемы, инфографика)
- Серийные посты (несколько частей одной темы)
- Больше кросс-ссылок

**Главный принцип:** Быть полезной, честной и регулярной. Давать не просто контент, а реальную ценность для подготовки к собеседованиям.
