# Понедельник: Теория про Стек

Стек: простая структура для сложных задач

🐥 Материал подойдет для новичков

1️⃣ Что такое стек?
Стек — это структура данных с принципом LIFO (Last In, First Out): последний пришёл — первый ушёл. Как стопка тарелок: можешь взять только верхнюю, а новую положить только сверху. В большинстве языков программирования стек не представлен в качестве отдельной структуры данных и реализуется с помощью других структур.

2️⃣ Основные операции
• push(x) — добавить элемент на вершину стека
• pop() — удалить и вернуть элемент с вершины
• peek() / top() — посмотреть верхний элемент без удаления
• isEmpty() — проверить, пуст ли стек

Других операций у классического стека нет.

При этом все операции должны работать за O(1). Это важно. Структуру данных с такими же операциями, но без константной сложности для них формально нельзя будет назвать стеком.

3️⃣ Вы точно встречали стек в жизни
• История браузера (кнопка «Назад»)
• Undo/Redo в редакторах
• Стек вызовов функций (call stack)

4️⃣ Классические задачи на собеседованиях
По моей статистике, задачи на стек встречаются в ~12% собеседований в крупных российских компаниях. Самые частые:
• Проверка валидности скобочных последовательностей
• Вычисление арифметических выражений (калькулятор)
• Монотонный стек (продвинутая тема для поиска следующего большего/меньшего элемента)

5️⃣ Когда использовать стек?
Если в задаче нужно:
• Обрабатывать вложенность (скобки, теги HTML)
• Отменять действия
• Запоминать путь обхода (DFS в графах/деревьях)
• Работать с «самым последним добавленным» элементом

6️⃣ Реализация
• Python: list с методами append() и pop(); структура deque в библиотеке collections
• Java: Stack<T> или Deque<T> (рекомендуется), но можно и ArrayList
• Go: срез с операциями append и обрезкой
