Часть 2: Бонусная модификация — порядок не важен

Теперь интервьюер говорит: «Отлично! А теперь измените решение так, чтобы порядок скобок был не важен».

Ключевое изменение: вместо одного стека используем три отдельных стека — для каждого типа скобок.

Почему это работает: нам не важен порядок скобок разного типа между собой, поэтому каждый тип скобок приходится рассматривать отдельно. Заметьте, как более расслабленное условие делает задачу сложнее — так бывает нередко. Как думаете, почему?

Минимальное изменение кода:
def isValidUnordered(s: str) -> bool:
    # ТРИ СТЕКА вместо одного
    stacks = {
        '(': [],  # стек для круглых скобок
        '[': [],  # стек для квадратных скобок
        '{': []   # стек для фигурных скобок
    }

    pairs = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in '([{':
            # Открывающая — кладём в соответствующий стек
            stacks[char].append(char)
        else:
            # Закрывающая — проверяем соответствующий стек
            matching_open = pairs[char]

            # Проверяем, что в нужном стеке есть элементы
            if not stacks[matching_open]:
                return False

            # Удаляем из соответствующего стека
            stacks[matching_open].pop()

    # Проверяем, что все три стека пусты
    return (len(stacks['(']) == 0 and
            len(stacks['[']) == 0 and
            len(stacks['{']) == 0)

Пошаговый пример для "([)]":

Позиция 0: '(' → stacks = {'(': ['('], '[': [], '{': []}
Позиция 1: '[' → stacks = {'(': ['('], '[': ['['], '{': []}
Позиция 2: ')' → ищем в стеке '(', он не пуст → pop → stacks = {'(': [], '[': ['['], '{': []}
Позиция 3: ']' → ищем в стеке '[', он не пуст → pop → stacks = {'(': [], '[': [], '{': []}

Все стеки пусты → true ✅

Сложность: O(n) времени, O(n) памяти (так же, как классическое решение)
