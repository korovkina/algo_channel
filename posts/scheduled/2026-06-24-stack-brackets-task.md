# Вторник: Условие задачи (утро)

🔥 Свежая задача из rtlab (Госуслуги)

**Сложность:** Easy

**Валидная скобочная последовательность с особым правилом**

Дана строка, состоящая только из символов `'('`, `')'`, `'['`, `']'`, `'{'`, `'}'`.

Определите, является ли эта строка валидной скобочной последовательностью.

**Особое условие:** Порядок скобок не важен. Важно только, чтобы после каждой открывающей скобки где-то в строке встречалась соответствующая ей закрывающая скобка.

**Требования:**
• Время: O(n)
• Память: O(n)

**Примеры:**

Вход: `"()"`
Выход: `true`

Вход: `"()[]{}"`
Выход: `true`

Вход: `"({[]})"`
Выход: `true`

Вход: `"([)]"`
Выход: `true`
_Объяснение: Порядок не важен. Есть открывающая `(` и закрывающая `)`. Есть открывающая `[` и закрывающая `]`. Все в порядке._

Вход: `"(("`
Выход: `false`
_Объяснение: Две открывающие скобки, но нет ни одной закрывающей._

Вход: `")("`
Выход: `false`
_Объяснение: Закрывающая скобка идёт раньше открывающей — это невалидно._

Вход: `"((]"`
Выход: `false`
_Объяснение: Две открывающие круглые скобки `(`, но только одна закрывающая квадратная `]`. Несоответствие типов._

**Подсказка:** ||Несмотря на то, что порядок не важен, нужно следить за балансом: в любой момент времени количество закрывающих скобок не должно превышать количество открывающих.||

# Вторник: Разбор задачи (через 3 часа)

**Разбор задачи из rtlab (Госуслуги)** из прошлого поста

**Паттерн:** Стек

**Ключевая идея:** Используем словарь (хэш-мапу) для подсчёта открывающих и закрывающих скобок каждого типа. Важно следить, чтобы в любой момент закрывающих скобок не было больше, чем открывающих.

**Алгоритм:**

1️⃣ Создаём два словаря: `open_count` и `close_count` для подсчёта открывающих и закрывающих скобок каждого типа.

2️⃣ Проходим по строке слева направо:
   • Если встретили открывающую скобку — увеличиваем счётчик в `open_count`
   • Если встретили закрывающую скобку — увеличиваем счётчик в `close_count`
   • **Важно:** После каждой закрывающей скобки проверяем, что закрывающих скобок этого типа не больше, чем открывающих. Если больше → возвращаем `false`

3️⃣ В конце проверяем, что количество открывающих и закрывающих скобок каждого типа совпадает.

**Код на Python:**

```python
def isValid(s: str) -> bool:
    # Словарь для сопоставления скобок
    pairs = {')': '(', ']': '[', '}': '{'}

    # Счётчики для каждого типа скобок
    open_count = {'(': 0, '[': 0, '{': 0}
    close_count = {')': 0, ']': 0, '}': 0}

    for char in s:
        if char in open_count:
            # Открывающая скобка
            open_count[char] += 1
        else:
            # Закрывающая скобка
            close_count[char] += 1
            # Проверяем, что закрывающих не больше открывающих
            matching_open = pairs[char]
            if close_count[char] > open_count[matching_open]:
                return False

    # Проверяем, что все типы скобок сбалансированы
    return (open_count['('] == close_count[')'] and
            open_count['['] == close_count[']'] and
            open_count['{'] == close_count['}'])
```

**Пошаговый пример для `"([)]"`:**

Позиция 0: `'('` → `open_count = {'(': 1, '[': 0, '{': 0}`
Позиция 1: `'['` → `open_count = {'(': 1, '[': 1, '{': 0}`
Позиция 2: `')'` → `close_count = {')': 1, ']': 0, '}': 0}`
   Проверка: `close_count[')'] (1) ≤ open_count['('] (1)` ✅
Позиция 3: `']'` → `close_count = {')': 1, ']': 1, '}': 0}`
   Проверка: `close_count[']'] (1) ≤ open_count['['] (1)` ✅

Финальная проверка: все типы сбалансированы → `true`

**Сложность:** O(n) времени, O(1) памяти (словари фиксированного размера)

——

**Важное отличие от классической задачи:** В обычной задаче на валидные скобки мы используем стек и проверяем, что закрывающая скобка соответствует последней открывающей. Здесь же порядок не важен, поэтому достаточно простого подсчёта.
