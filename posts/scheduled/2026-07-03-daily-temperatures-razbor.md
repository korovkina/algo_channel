Разбор Daily Temperatures из поста про лондонскую жару

Вы дали достаточно 👀, так что собираю мозги в кучу, как и обещала. Кстати, потепление в Лондоне закончилось, так и не начавшись, — а вот в задаче мы его сейчас найдём за O(n).

Паттерн: монотонный стек (убывающий)

Ключевая идея: идём по дням слева направо и держим в стеке индексы дней, которые всё ещё ждут своего потепления. Температуры по этим индексам убывают — как только приходит день теплее вершины стека, он «раздаёт ответы» всем дням из стека, которые холоднее его.

Алгоритм:

1️⃣ Создаём массив `answer` из нулей (ноль — это и есть ответ для дней, которые потепления не дождутся) и пустой стек для индексов.

2️⃣ Для каждого дня `i`: пока стек не пуст и температура на его вершине ниже сегодняшней — снимаем индекс `j` со стека и записываем `answer[j] = i - j`. Сегодняшний день и есть их долгожданное потепление.

3️⃣ Кладём `i` в стек — теперь он сам ждёт тепла.

Код на Python:

```python
def dailyTemperatures(temperatures: list[int]) -> list[int]:
    n = len(temperatures)
    answer = [0] * n
    stack = []  # индексы дней, которые ждут потепления

    for i, t in enumerate(temperatures):
        # Сегодня теплее, чем на вершине стека?
        # Раздаём ответы всем, кто дождался
        while stack and temperatures[stack[-1]] < t:
            j = stack.pop()
            answer[j] = i - j
        stack.append(i)

    return answer
```

Пошагово на примере из условия `[73, 74, 75, 71, 69, 72, 76, 73]`:

• День 0 (73°): стек пуст → кладём. Стек: `[0]`
• День 1 (74°): 74 > 73 → день 0 дождался, `answer[0] = 1`. Стек: `[1]`
• День 2 (75°): 75 > 74 → `answer[1] = 1`. Стек: `[2]`
• Дни 3 (71°) и 4 (69°): холоднее вершины → просто ждут. Стек: `[2, 3, 4]`
• День 5 (72°): раздаёт сразу двоим! 72 > 69 → `answer[4] = 1`; 72 > 71 → `answer[3] = 2`. А 75 теплее — ждёт дальше. Стек: `[2, 5]`
• День 6 (76°): 76 > 72 → `answer[5] = 1`; 76 > 75 → `answer[2] = 4`. Стек: `[6]`
• День 7 (73°): холоднее → ждёт. Стек: `[6, 7]`

Дни 6 и 7 остались в стеке — им потепление не светит, у них честные нули.

Ответ: `[1, 1, 4, 2, 1, 1, 0, 0]` ✅

Сложность: O(n) времени — каждый индекс попадает в стек и покидает его ровно один раз, поэтому никакого O(n²), хотя цикл внутри цикла и смущает. O(n) памяти на стек в худшем случае (температуры строго убывают — как раз наш лондонский сценарий).

——

Монотонный стек — это целое семейство задач вида «найди ближайший больший/меньший элемент»: Next Greater Element, площадь максимального прямоугольника в гистограмме. В моих закромах реальных собесов такое тоже водится: в Авито, например, давали сумму минимумов всех подмассивов — тот же паттерн, вид сбоку.
