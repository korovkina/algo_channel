## Текст для публикации:

❌ Частая ошибка: "Сортировка решает всё"

На этой неделе мы говорим про сортировки, и я хочу показать ошибку, которую вижу на собесах постоянно: кандидат видит задачу про массив и первым делом его сортирует, даже когда есть решение лучше.

Разберем на реальном примере.

Задача: Missing Number (LeetCode 268)

Дан массив из n чисел в диапазоне [0, n]. Одно число отсутствует. Найти его.

Пример:
- Вход: [3, 0, 1]
- Выход: 2

Вход: [0, 1]
Выход: 2

Что делает кандидат (типичное первое решение):

def missing_number(nums):
    nums.sort()  # O(n log n)
    for i in range(len(nums)):
        if nums[i] != i:
            return i
    return len(nums)

Логика: "Отсортирую массив, тогда каждое число будет на своей позиции (0 на месте 0, 1 на месте 1...), и я сразу увижу где пропуск!"

Решение работает. Но сложность O(n log n).

——

А теперь вопрос от интервьюера: "Можете лучше?"

И тут начинается ступор.

Правильные решения за O(n):

1️⃣ Математическая формула (сумма арифметической прогрессии):

def missing_number(nums):
    n = len(nums)
    expected_sum = n * (n + 1) // 2
    actual_sum = sum(nums)
    return expected_sum - actual_sum

Сложность: O(n) времени, O(1) памяти

2️⃣ XOR (битовая магия):

def missing_number(nums):
    result = len(nums)
    for i in range(len(nums)):
        result ^= i ^ nums[i]
    return result

Сложность: O(n) времени, O(1) памяти

Почему XOR работает? Потому что x ^ x = 0, и если XOR-ить все числа от 0 до n и все числа из массива, все парные числа обнулятся, останется только пропущенное.

3️⃣ Set (если не важна память):

def missing_number(nums):
    full_set = set(range(len(nums) + 1))
    return list(full_set - set(nums))[0]

Сложность: O(n) времени, O(n) памяти

——

В чём ошибка?

Сортировка - это не плохо. Но когда видите задачу с массивом:

1. Остановитесь и подумайте: а точно ли нужна сортировка?
2. Есть ли в задаче математическая структура? (последовательные числа, диапазон, сумма)
3. Можно ли решить за O(n) через хеш-мапу, set, формулу или битовые операции?
4. Сортировка даёт O(n log n) - это часто не оптимально

Правило:

Сортировка - мощный инструмент, но не универсальное решение.
Прежде чем сортировать, спроси себя:
"Могу ли я решить за O(n) без сортировки?"

——

На собесах это очень важно. Интервьюер хочет видеть, что вы:
✅ Анализируете условия задачи
✅ Видите паттерны (последовательность → формула)
✅ Не применяете один и тот же подход ко всем задачам

Кстати, эту задачу дают в Яндексе, Озоне, Т-банке. Если на собесе начнёте с сортировки, скорее всего услышите: "А можно лучше?"
