## Пост 2: Разбор (через 3 часа)

### Текст для публикации:

Разбор задачи из Ozon из поста выше

Паттерн: Counting Sort (сортировка подсчётом)

Ключевая идея: когда диапазон значений ограничен, можем посчитать частоту каждого значения, а затем восстановить отсортированный массив.

Почему стандартная сортировка не подойдет?
Массив до 10^8 элементов — это 100 миллионов чисел. Даже с O(n log n) это будет ~2,6 миллиарда операций. Здесь мы можем справиться лучше.

Алгоритм Counting Sort:
1️⃣ Создаём массив частот размером 101 (от 0 до 100)
2️⃣ Подсчитываем сколько раз встречается каждое число
3️⃣ Восстанавливаем отсортированный массив: проходим по массиву частот и добавляем каждое число нужное количество раз

Код на Python:
def counting_sort(nums):
    # Диапазон значений: 0-100
    max_val = 100
    count = [0] * (max_val + 1)

    # Шаг 1: Подсчитываем частоты
    for num in nums:
        count[num] += 1

    # Шаг 2: Восстанавливаем отсортированный массив
    result = []
    for value in range(max_val + 1):
        result.extend([value] * count[value])

    return result
Пошаговый пример для [5, 2, 8, 2, 1, 5, 100, 0]:
После подсчёта:
count[0] = 1
count[1] = 1
count[2] = 2
count[5] = 2
count[8] = 1
count[100] = 1

Восстановление:
Добавляем 0 один раз: [0]
Добавляем 1 один раз: [0, 1]
Добавляем 2 два раза: [0, 1, 2, 2]
Добавляем 5 два раза: [0, 1, 2, 2, 5, 5]
Добавляем 8 один раз: [0, 1, 2, 2, 5, 5, 8]
Добавляем 100 один раз: [0, 1, 2, 2, 5, 5, 8, 100]
Сложность:
- Время: O(n + k) где n = размер массива, k = 101 (диапазон)
- Память: O(k) = O(101) = O(1) константная

Ощутите разницу: для 10^8 элементов это всего ~100 миллионов операций вместо 2,6 миллиардов.

Когда использовать Counting Sort?
✅ Диапазон значений ограничен и известен
✅ Диапазон не слишком большой (k << n или k приемлемо)
✅ Значения - целые числа
