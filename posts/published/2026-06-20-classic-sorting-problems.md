🔥 Задача 1: 3Sum

Условие: Дан массив чисел. Найти все уникальные тройки, которые в сумме дают 0.

Пример:
- Вход: [-1, 0, 1, 2, -1, -4]
- Выход: [[-1, -1, 2], [-1, 0, 1]]

Паттерн: Сортировка + Two Pointers

Алгоритм:
1. Сортируем массив по возрастанию
2. Фиксируем первый элемент тройки (проходим циклом)
3. Для оставшейся части используем два указателя:
   - left — сразу после зафиксированного элемента
   - right — в конце массива
4. Считаем сумму трёх элементов:
   - Если сумма = 0 → нашли тройку, сдвигаем оба указателя
   - Если сумма < 0 → двигаем left вправо (увеличиваем сумму)
   - Если сумма > 0 → двигаем right влево (уменьшаем сумму)
5. Пропускаем дубликаты на всех уровнях

Сложность: O(n²) времени, O(1) памяти

Почему сортировка здесь ключ к решению:
- Без сортировки пришлось бы перебирать все тройки → O(n³)
- После сортировки two pointers работает за O(n) для каждого зафиксированного элемента
- Сортировка позволяет легко пропускать дубликаты

Похожие задачи: 3Sum Closest, 4Sum

——

🔥 Задача 2: Merge Intervals

Условие: Дан массив интервалов [start, end]. Объединить все перекрывающиеся интервалы.

Пример:
- Вход: [[1,3], [2,6], [8,10], [15,18]]
- Выход: [[1,6], [8,10], [15,18]]

Паттерн: Сортировка по началу интервала + линейный проход

Алгоритм:

1. Сортируем интервалы по начальной точке (start)
2. Берём первый интервал как текущий
3. Проходим по остальным интервалам:
   - Если текущий интервал пересекается со следующим → объединяем их (расширяем end)
   - Если не пересекается → сохраняем текущий интервал, переходим к следующему
4. Сохраняем последний интервал

Сложность: O(n log n) времени (сортировка), O(n) памяти

Почему сортировка здесь ключ к решению:
- Без сортировки пришлось бы сравнивать каждый интервал с каждым → O(n²)
- После сортировки все потенциально пересекающиеся интервалы гарантированно идут подряд
- Это позволяет решить задачу за один линейный проход → O(n)

Задачи на интервалы очень часто решаются с предварительной сортировкой. Вот еще примеры: Insert Interval, Non-overlapping Intervals, Meeting Rooms II

——

🔥 Задача 3: Group Anagrams

Условие: Дан массив слов. Сгруппировать все анаграммы вместе.

Пример:
- Вход: ["eat", "tea", "tan", "ate", "nat", "bat"]
- Выход: [["eat","tea","ate"], ["tan","nat"], ["bat"]]

Паттерн: Сортировка символов + HashMap

Алгоритм:

1. Для каждого слова:
   - Сортируем его символы (например, "eat" → "aet")
   - Используем отсортированную строку как ключ в хеш-мапе
   - Добавляем исходное слово в список по этому ключу
2. Все анаграммы после сортировки символов дают одинаковый ключ и группируются автоматически

Сложность: O(n × k log k), где n — количество слов, k — средняя длина слова

Почему сортировка здесь ключ к решению:
- Без сортировки нет очевидного способа определить, являются ли два слова анаграммами за O(k log k)
- Сортировка символов превращает все анаграммы в идентичную строку → создаёт уникальный ключ для группировки
- Это самое элегантное и эффективное решение для задач на анаграммы

Бонусный вопрос: хотя решение с сортировкой считается классическим, эту задачу можно решить и без нее. Как?
