# Условие: Merge Sorted Array (запланировано на 10 июля, утро, 07:00-12:00)

Штош, время пришло, разбираю топ-1 задачу с собесов по моей личной статистике. Видели ее в Химках в Яндексе, Авито, ВКонтакте, в общем где только не. Популярность задачи меня не удивляет: выглядит она беззубо, а на деле присутствует нюанс.

Сложность: Easy (с плюсом)

Условие:
Даны два отсортированных по возрастанию массива nums1 и nums2 длиной m и n. Нужно слить их в один отсортированный массив.

Хитрость в том, куда складывать результат: nums1 уже имеет размер m + n: первые m ячеек заняты элементами, а последние n забиты нулями-заглушками как раз под элементы nums2. Слить нужно прямо в nums1, без создания нового массива.

Пример:

- Вход: nums1 = [1, 2, 3, 0, 0, 0], m = 3, nums2 = [2, 5, 6], n = 3
- Выход: [1, 2, 2, 3, 5, 6]

Ещё пример:

- Вход: nums1 = [4, 5, 6, 0, 0, 0], m = 3, nums2 = [1, 2, 3], n = 3
- Выход: [1, 2, 3, 4, 5, 6]

(специально дала случай, где весь nums2 меньше всего nums1 — на нём наивные решения особенно больно спотыкаются)

Требования:
- Время: O(m + n)
- Память: O(1) — то есть in-place, без второго массива

Подсказка: очевидное направление слияния здесь работает против вас. Попробуйте зайти с другой стороны.
