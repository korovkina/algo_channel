## Среда: Частая ошибка
**Предложение:** "Почему нельзя использовать обычный массив вместо стека?"

**Структура поста:**
1. Хук: ❌ <strong>Частая ошибка: "А зачем стек? Можно же просто массив"</strong>
2. Маркер: 🐥 Материал для новичков
3. Диалог:
   - Кандидат: "Вместо стека я просто создам список и буду добавлять элементы в конец"
   - Интервьюер: "А как вы будете искать последний элемент?"
   - Кандидат: "Пройду по всему списку..."
4. Объяснение проблемы:
   - Стек гарантирует O(1) для всех операций
   - Массив без индекса последнего элемента потребует O(n) для поиска
   - Стек — это абстракция с чёткими ограничениями
5. Правильный подход:
   - Python: list с append()/pop()
   - Java: Deque<T> (предпочтительнее Stack<T>)
   - Go: slice с операциями append и срезом
6. Общее правило: "Используй правильную абстракцию для задачи"
7. CTA + хештеги: #ошибка #стек

**Почему эта тема:** Новички часто не понимают разницу между структурой данных и её реализацией. Этот пост поможет понять, зачем нужны абстракции.
