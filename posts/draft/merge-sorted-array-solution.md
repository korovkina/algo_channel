# Разбор: Merge Sorted Array (запланировано на 10 июля, день/следующий день по итогам опроса)

Разбор задачи про слияние отсортированных массивов из прошлого поста

Паттерн: два указателя (но с неожиданной стороны)

Сначала о граблях, ради которых я эту задачу и выбрала.

Ловушка. Первое, что просится: два указателя в начале массивов, сравниваем и пишем меньший в `nums1` слева направо. И тут же катастрофа: в `nums1` слева лежат живые элементы, которые мы ещё не прочитали. Записывая ответ в начало, мы затираем то, что нам самим понадобится через шаг. На первом примере может даже повезти, а вот на `nums1 = [4,5,6,0,0,0]`, `nums2 = [1,2,3]` первая же единица из `nums2` затрёт четвёрку, и всё поехало. Именно поэтому я просила проверить код на втором примере.

Обойти это можно, заведя копию `nums1`, но тогда прощай `O(1)` памяти, а её с нас требуют.

Ключевая идея: заполняем `nums1` с конца, от большего к меньшему. В хвосте `nums1` лежат нули-заглушки, пустое место, которое не жалко перезаписывать. Пока мы движемся справа налево, мы всегда пишем в ячейку, которую уже прочитали или которая была заглушкой. Затирать живые данные физически невозможно. Одна смена направления, и проблема исчезает.

Алгоритм:

1️⃣ Три указателя: `i = m - 1` (конец реальных данных `nums1`), `j = n - 1` (конец `nums2`), `k = m + n - 1` (конец всего `nums1`, туда пишем).

2️⃣ Пока в `nums2` есть элементы: сравниваем `nums1[i]` и `nums2[j]`, больший кладём в `nums1[k]`, двигаем соответствующий указатель и `k` влево.

3️⃣ Когда `nums2` кончился, всё готово: оставшиеся элементы `nums1` уже стоят на своих местах.

Код на Python:

```python
def merge(nums1, m, nums2, n):
    i = m - 1          # конец реальных данных nums1
    j = n - 1          # конец nums2
    k = m + n - 1      # куда пишем — самый конец nums1

    while j >= 0:      # пока в nums2 есть что вставлять
        if i >= 0 and nums1[i] > nums2[j]:
            nums1[k] = nums1[i]
            i -= 1
        else:
            nums1[k] = nums2[j]
            j -= 1
        k -= 1
```

Обратите внимание на условие цикла: `while j >= 0`, а не по обоим указателям. Если раньше кончился `nums2`, работа закончена, элементы `nums1` и так на местах. Если кончился `nums1`, остаток `nums2` спокойно допишется через ветку `else`.

Пошагово на `nums1 = [1,2,3,0,0,0]`, `nums2 = [2,5,6]`:

• `i=2 (3)`, `j=2 (6)`: 6 больше → `nums1[5]=6`. Массив: `[1,2,3,0,0,6]`
• `i=2 (3)`, `j=1 (5)`: 5 больше → `nums1[4]=5`. `[1,2,3,0,5,6]`
• `i=2 (3)`, `j=0 (2)`: 3 больше → `nums1[3]=3`, двигаем `i`. `[1,2,3,3,5,6]`
• `i=1 (2)`, `j=0 (2)`: равны, берём из `nums2` → `nums1[2]=2`, двигаем `j`. `[1,2,2,3,5,6]`
• `j = -1` → выходим. Первые `[1,2]` не тронули, они и так на месте.

Ответ: `[1, 2, 2, 3, 5, 6]` ✅

Сложность: `O(m + n)` времени, каждый элемент перекладываем ровно один раз. `O(1)` памяти, пишем внутрь `nums1`, никаких копий.

——

Слияние двух отсортированных, это фундамент, на котором стоит куда больше, чем кажется. Merge Sort целиком построен на этом шаге. Есть родная сестра этой задачи на связных списках (Merge Two Sorted Lists), там трюка «с конца» нет, потому что список назад не пройти, зато выручает dummy-узел. По моей статистике реальных собесов слияние в том или ином виде, одна из самых частых задач: Яндекс, Авито, ВК гоняют её регулярно. Разберётесь с in-place версией, считайте, что закрыли целое семейство.
